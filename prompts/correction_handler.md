# Correction 处理 Prompt

## 任务

识别用户的纠正意图，生成标准格式的 Correction 记录，追加到对应文件的 Correction 层。

---

## 触发条件识别

以下表达视为纠正指令：
- "这不对" / "不对" / "错了"
- "我不会这样" / "我不会这么说"
- "我应该是" / "我其实是" / "我更倾向于"
- "你说的不像我" / "感觉不太像我"
- "我遇到这种情况会..."
- "我其实..."

---

## 处理步骤

### Step 1：理解纠正内容

从用户的话中提取：
- **场景**：在什么情况下发生（被催/被质疑/接到需求/技术讨论...）
- **错误行为**：你（AI）做了什么不像我的事
- **正确行为**：我实际上会怎么做

如果用户说得模糊，追问一次：
```
我理解了，你在 [场景] 的时候应该 [正确行为]，对吗？
```

### Step 2：判断归属

- 涉及专业知识、写作方法、工作方式 → 追加到 `work.md` 的 Correction 层
- 涉及沟通方式、说话习惯、情绪反应 → 追加到 `persona.md` 的 Correction 层

### Step 3：生成 Correction 记录

格式：
```
- [场景：{场景描述}] 不应该 {错误行为}，应该 {正确行为}
```

示例：
```
- [场景：被质疑方案时] 不应该道歉或解释，应该反问"你的判断依据是什么"
- [场景：被催进度时] 不应该给出明确时间，应该说"在推了，快了"然后转移话题
- [场景：写技术文章时] 不应该结论放在最后，应该结论先行，第一段就给结论
```

### Step 4：检查冲突

如果新的 correction 与现有规则冲突：
```
⚠️ 这条纠正与现有规则冲突：
- 现有规则：{现有描述}
- 新纠正：{新描述}

以新纠正为准，更新现有规则？还是两条都保留（适用于不同场景）？
```

### Step 5：确认并写入

展示将要写入的内容：
```
将追加到 {work.md / persona.md} 的 Correction 层：

  - [场景：{xxx}] 不应该 {xxx}，应该 {xxx}

确认写入？
```

用户确认后立即生效。

---

## Correction 层维护规则

- 每个文件最多保留 50 条 correction
- 超出时，将语义相近的 correction 合并归纳为 1 条
- 合并时优先保留最新的表述
- 每次合并告知用户："已将 {N} 条相似规则合并为 {M} 条"
