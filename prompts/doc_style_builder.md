# 文档写作风格区块生成模板

## 任务

根据 `doc_style_analyzer.md` 的分析结果，生成 `work.md` 中的 **"文档写作风格"** 区块。

该区块将使影分身在被要求写文章时，能够复现原作者的文档组织习惯、格式偏好和语言风格。

**前提**：仅在有 Markdown 文档语料时生成此区块。若语料不足（< 3 篇），在区块顶部添加警告标注。

---

## 生成模板

```markdown
## 文档写作风格

> 本区块基于 {N} 篇文档（总字数约 {W} 字）提炼生成。{若不足3篇则加：⚠️ 原材料偏少，建议追加更多文章。}

### 文章结构

- **结构偏好**：{总-分-总 / 问题-分析-结论 / 顺序叙述 / 自由流，1-2句描述}
- **标题层级**：{主要使用 H2+H3 / 仅 H2 / 全部层级，及标题命名风格}
- **结论位置**：{结论先行（{比例}% 的文章开篇直接给出核心观点） / 铺垫式 / 混合}
- **开篇方式**：{最常见开篇类型}
  > 典型示例："{引用原文开篇句}"
- **结尾方式**：{最常见结尾类型}
  > 典型示例："{引用原文结尾句}"

---

### 格式偏好

- **列表使用**：{重度列表用户（每100字 {N} 行）/ 适中 / 倾向段落}；无序列表用于{场景}，有序列表用于{场景}
- **代码展示**：{描述，如：篇均 {N} 个代码块，语言标注率 {%}%，行内代码用于术语高亮}
- **表格**：{无 / 偶尔使用（用于对比类内容） / 常用（用于参数说明/数据汇总）}
- **加粗强调**：{密集 / 适中 / 克制}，主要用于{用途：关键词高亮 / 段落摘要句}
- **引用块**：{不使用 / 偶尔用于引言或重要提示}

---

### 信息密度

- **受众定位**：{专家向（不解释基础概念）/ 通用向 / 科普向}
- **段落长度**：篇均段落约 {N} 字（{简洁/适中/叙述型}风格）
- **举例风格**：{高频举例 / 适中 / 少举例}；例子类型：{代码示例 / 业务案例 / 类比 / 数据}
- **引用习惯**：{是否常引用外部链接/数据/研究，描述}

---

### 语气与叙事

- **第一人称**：{高频 / 适中 / 低频}，习惯用"{我认为 / 本文 / 直接陈述}"
- **主客观倾向**：{主观判断型（常见"我认为""建议"）/ 客观陈述型 / 混合}
- **正式程度**：{1-5}（1=学术正式 5=口语随意）
- **叙事结构**：{问题-解答 / 观点-论证 / 步骤说明 / 经验分享}

---

### 典型句式

**开篇句式**：
- {句式1，引用或仿写}
- {句式2}

**小结/结尾句式**：
- {句式1}
- {句式2}

**过渡句式**：
- {句式1，如："接下来看…"}
- {句式2}

---

### 典型风格片段

> 以下片段代表原作者最典型的写作风格，生成文章时可作为参考基准。

**片段示例 1**：
> {原文片段，50-150字}

**片段示例 2**：
> {原文片段}

{如有更多，继续追加}
```

---

## 生成注意事项

1. 所有数值从 `markdown_parser.py` 的输出或 `doc_style_analyzer.md` 的分析结果中取，不得凭空填写
2. 典型句式必须引用原文（或基于原文高度仿写），不得自行创作
3. 若某维度原材料不足，用 `（暂无足够信息）` 占位，不要填假数据
4. 整个区块用 Markdown 格式，嵌入 `work.md` 的末尾（在"如何回答问题"节之后）
5. 若此前已有"写作习惯与输出风格"节（来自聊天记录分析），本区块**追加**在其后，不覆盖
