# 文档写作风格分析 Prompt

## 任务

你将收到：
1. 用户的基础信息（代号、背景、个性标签等，来自 Step 1）
2. `markdown_parser.py` 输出的文档结构摘要（数值统计）
3. 原始 Markdown 文档内容（若文档数量 ≤ 5 篇，全文提供；超过 5 篇则提供每篇开头 + 结尾各 300 字及摘要统计）

从中提取 **{name}** 的**文档写作风格**，用于构建影分身的文档写作层（Doc Style）。

**原则：只提取有实质证据的结论，没有依据的维度标注"原材料不足"，不要推断。**

> **数据充分性门槛**：若文档数 < 3 篇，在分析结果最前面注明：
> `⚠️ 原材料不足（仅 N 篇文档），以下分析仅供参考，建议追加更多文章以提高准确性。`

---

## 分析维度

### 1. 文章结构偏好

根据统计数据和原文，分析：

**标题层级使用模式**：
- 主要使用哪几级标题（仅 H2 / H2+H3 / 全部层级）
- 标题命名风格（陈述式 / 疑问式 / 动宾结构 / 数字清单 / 如何/为什么）
- 标题平均字数（短标题 <6 字 / 中等 6-15 字 / 长标题 >15 字）

**内容组织逻辑**：
- 是否习惯结论先行（开篇直接给出核心观点）
- 主体内容的典型组织方式（总-分-总 / 问题-分析-结论 / 顺序叙述 / 自由流）
- 是否有可识别的固定文章模板

**开篇 / 结尾模式**：
- 最常见的开篇方式（疑问引入 / 背景铺垫 / 直接陈述）
- 最常见的结尾方式（总结要点 / 行动号召 / 开放式结尾）

```
输出格式：
标题层级：[描述]
标题命名规律：[描述]
结论先行：[是/否/混合]
内容组织：[描述]
开篇方式：[最常见类型] + [典型示例句（引用原文）]
结尾方式：[最常见类型] + [典型示例句（引用原文）]
```

---

### 2. 格式使用风格

**列表偏好**：
- 列表密度（每100字列表行数）→ 判断：重度列表用户 / 适度使用 / 倾向段落
- 有序 vs 无序列表的使用场景（无序=并列事项，有序=步骤/排名）
- 嵌套列表的使用深度和频率

**代码 / 技术内容展示**：
- 代码块平均每篇数量（无 / 少量 1-2 / 中等 3-5 / 密集 5+）
- 代码块语言标注率（是否每个代码块都标注语言）
- 行内代码的使用密度（用于术语/命令高亮的习惯）

**表格使用**：
- 使用频率（无 / 偶尔 / 常用）
- 什么场景触发表格（对比 / 参数说明 / 数据汇总）

**强调使用**：
- 加粗的使用密度（每千字次数）→ 密集 >5 / 适中 2-5 / 克制 <2
- 加粗的用途（关键词高亮 / 段落摘要句 / 无明显规律）
- 引用块（blockquote）的使用场景

```
输出格式：
列表偏好：[重度/适度/倾向段落]，有序用于[场景]，无序用于[场景]
代码展示：[描述] 语言标注率[百分比]
表格：[无/偶尔/常用]，用于[场景]
加粗：[密集/适中/克制]，主要用于[用途]
```

---

### 3. 信息密度与受众定位

**受众假设**：
- 是否假设读者有背景知识（专家向 / 通用向 / 科普向）
- 术语是否附带解释（前置定义 / 行文中解释 / 直接使用不解释）
- 段落平均字数（<80 字=信息密度高、结构分散；80-200 字=适中；>200 字=叙述型）

**举例方式**：
- 举例频率（高频 / 适中 / 少）
- 例子类型（代码示例 / 现实业务案例 / 类比 / 数据/实验）

**数据与引用**：
- 是否常引用外部数据、研究或链接
- 图片/插图的使用频率

```
输出格式：
受众定位：[专家向/通用向/科普向]
术语处理：[描述]
段落风格：[简洁/适中/叙述型]
举例风格：[描述] 例子类型：[类型列表]
引用习惯：[描述]
```

---

### 4. 语气与叙事风格

**第一人称使用**：
- 是否频繁使用"我"/"我们"（每千字次数 > 5 为高频，1-5 为适中，<1 为低频）
- 常用称谓方式（我认为 / 本文 / 作者 / 不用称谓直接陈述）

**主客观比例**：
- 是否以主观判断为主（"我认为" / "建议" / "最好"）
- 还是以客观陈述为主（第三人称 / 数据说话）

**写作语气**：
- 正式程度（1=学术正式 / 2=专业 / 3=平实 / 4=口语化 / 5=随意）
- 是否带有个人情感/评价（中性 / 带有明显立场 / 有情绪色彩）
- 常用叙事结构（问题-解答 / 观点-论证 / 步骤说明 / 经验分享）

```
输出格式：
第一人称：[高频/适中/低频]，习惯用"[实际用词]"
主客观：[主观判断型/客观陈述型/混合]
正式程度：[1-5]
叙事结构：[描述]
```

---

### 5. 高频写作模式（可直接用于生成）

从原文中提取 3-5 个最能代表其风格的**典型片段**，标注来源文件名：

```
片段 1（来自：{文件名}）：
"{原文片段，50-150字}"
风格特征：[这个片段体现了什么写作习惯]

片段 2（来自：{文件名}）：
"{原文片段}"
风格特征：[...]

...
```

同时提取：
- 最典型的 **开篇句式**（2-3 个示例，引用原文）
- 最典型的 **结尾/小结句式**（2-3 个示例）
- 最典型的 **过渡句式**（如"接下来…"/"那么问题来了…"）

---

## 输出要求

- 语言：中文
- 没有原文依据的维度：标注 `（原材料不足）`
- 有原文依据的结论：引用原话（加引号）
- 输出结果直接用于生成 `doc_style_builder.md` 的"文档写作风格"区块，要求具体可执行
- 不要写"可能""倾向于"这类模糊表述，只写有把握的结论
