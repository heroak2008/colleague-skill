# 增量 Merge Prompt

## 任务

你将收到：
1. 现有的 `work.md` 内容
2. 现有的 `persona.md` 内容
3. 新的原材料内容（聊天记录、文章、笔记等）

你的任务是判断新内容应该更新哪个部分，并输出增量更新内容。

**原则：只追加增量，不覆盖已有结论。如有冲突，输出冲突提示让用户决定。**

---

## Step 1：分类判断

将新内容中的每条信息归类：

| 信息类型 | 归入 |
|---------|------|
| 专业知识、领域技术、解题方法、思维框架 | → work.md |
| 知识结论、经验判断、踩过的坑 | → work.md |
| 写作习惯、文章结构、输出风格偏好（来自聊天/笔记） | → work.md |
| **文档写作风格（来自 Markdown 文件解析）** | → work.md 的"文档写作风格"区块 |
| 沟通风格、口头禅、表达习惯 | → persona.md |
| 决策行为、人际行为、情绪模式 | → persona.md |
| 两者都有 | → 分别归入 |

---

## Step 2：检查冲突

对比新内容与现有内容：

- 如果新内容**补充**了现有信息（增加了新细节）→ 直接追加
- 如果新内容**确认**了现有信息 → 忽略（不重复写）
- 如果新内容**与现有信息矛盾** → 输出冲突提示：

```
⚠️ 发现冲突：
- 现有：{现有描述}
- 新发现：{新内容描述}
- 来源：{文件名/时间}

建议：[保留现有 / 更新为新内容 / 两者都保留并标注时间]
请用户决定。
```

---

## Step 3：生成更新 Patch

对 `work.md` 的更新，输出格式：
```
=== work.md 更新 ===

[追加到"专业领域/细分方向"节]
- {新内容}

[追加到"经验与知识结论"节]
- {新知识结论}

[无更新] 或 [以上章节有更新]
```

对 `persona.md` 的更新，输出格式：
```
=== persona.md 更新 ===

[追加到"Layer 2/口头禅与高频词"节]
- 新口头禅："{xxx}"

[追加到"Layer 4/同行/平级"节]
- {新行为描述}

[无更新] 或 [以上章节有更新]
```

---

## Step 3b：文档写作风格区块增量更新

**仅当新内容为 Markdown 文档（由 `markdown_parser.py` 解析）时执行此步骤。**

对比新文档的分析结果与现有"文档写作风格"区块：

### 典型句式更新
- 新发现的开篇句式 → **追加**到"典型句式-开篇句式"列表
- 新发现的结尾/小结句式 → **追加**到"典型句式-小结/结尾句式"列表
- 新发现的过渡句式 → **追加**到"典型句式-过渡句式"列表

### 格式偏好更新
- 若新数据**补充**了现有描述（如：发现了更多代码块语言） → 直接追加
- 若新数据**与现有描述矛盾**，触发冲突提示的判断标准：
  - 列表密度分类变化（如：`<1` 变为 `>3`，即"倾向段落"变为"重度列表用户"）
  - 结论先行比例差距 ≥ 40%（如：现有 20%，新数据 80%）
  - 正式程度评级相差 ≥ 2 档（如：现有 2，新数据 4）
  - 受众定位类别变化（如：专家向 → 科普向）

满足上述任一条件时，输出冲突提示：

```
⚠️ 文档风格冲突：
- 现有：{现有描述}
- 新发现：{新数据描述}（来自：{文件名}）

建议：[保留现有 / 更新为新数据 / 两者都保留并标注"不同时期风格"]
请用户决定。
```

### 典型风格片段更新
- 新文档中的典型片段 → 若与现有片段风格差异明显，**追加**到"典型风格片段"（最多保留 5 段）

输出格式：
```
=== work.md "文档写作风格"区块更新 ===

[追加到"典型句式-开篇句式"]
- {新句式}

[追加到"典型风格片段"]
> {新片段}

[无更新] 或 [以上内容有更新]
```

---

## Step 4：生成更新摘要

向用户展示：
```
本次更新摘要：
- work.md：追加了 {N} 条新信息（{简要描述}）
- persona.md：追加了 {N} 条新信息（{简要描述}）
- 文档写作风格：{追加了 {N} 条句式/片段 / 无更新 / 发现 {N} 处冲突待确认}
- 发现 {N} 处冲突，需要你确认（见上方）

版本将从 {vN} 升级到 {vN+1}。
确认应用更新？
```
