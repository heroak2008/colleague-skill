# 知识与能力分析 Prompt

## 任务

你将收到 **{name}** 的原材料（聊天记录、文章、笔记等）。
从中提取其知识体系与工作能力，用于构建影分身的知识层（Work Skill）。

**原则：只提取有实质内容的信息，不要推断，有依据才写，没有就标注"原材料不足"。**

---

## 通用提取维度（所有职业适用）

### 1. 专业领域

从原材料中识别：
- 核心擅长领域（技术/产品/研究/写作等）
- 常提到的细分方向、项目类型
- 频繁使用的术语、概念
- 明显熟悉 vs 明显不熟悉的话题

```
输出格式：
核心领域：[描述]
细分方向：[列表]
高频术语：[列表]
```

### 2. 知识体系与思维框架

从讨论、回答、文章中提取：
- 他解决问题的思维步骤（先做什么，再做什么）
- 常用的分析框架、模型、方法论
- 如何拆解复杂问题
- 对哪类问题有独特的判断视角

```
输出格式：
解题步骤：[描述]
常用框架：[列表]
独特视角：[描述]
```

### 3. 写作习惯与输出风格

- 文章/回复的结构偏好（结论先行 vs 铺垫式）
- 用表格/列表/流程图/纯文字
- 内容详细程度（极简/适中/详尽）
- 引用来源的习惯（是否常引用数据/文献/案例）

```
输出格式：
文章结构：[描述]
格式偏好：[描述]
详细程度：[极简/适中/详尽]
引用习惯：[描述]
```

### 4. 经验与知识结论

本人明确表达的判断、踩过的坑、独到观点（直接引用原话）：

```
- "[原话或总结]"
- "[原话或总结]"
```

## 职业专项提取

根据 {name} 的职业背景，重点提取对应维度：

---

### 🖥️ 技术/工程师

**技术标准**：
- 技术栈（语言、框架、工具）
- 对代码质量/架构的看法
- 调试/排查问题的步骤
- 踩过的技术坑和反思

**回答问题的方式**：
- 技术问题如何解释（直接给结论 vs 先讲原理）
- 遇到不确定的技术问题如何回应

---

### 📝 写作者/内容创作者

**写作模式**：
- 选题偏好（什么类型的话题最感兴趣）
- 文章结构模板（标题→开头→核心论点→案例→结尾）
- 语言风格（正式/口语/专业/通俗）
- 常用开头/结尾句式

**内容观点**：
- 有哪些反复出现的核心主张
- 哪些话题是其"主场"

---

### 📱 产品/运营

**需求与问题处理**：
- 如何定义问题、找关键指标
- 优先级判断方式
- 推进事项的惯用方法

**输出物风格**：
- 文档详细程度
- 数据引用频率
- 沟通/汇报的惯用格式

---

### 🤖 研究者/分析师

**研究方法**：
- 如何提出研究问题
- 常用分析框架
- 结论呈现方式（结论先 vs 数据先）

**知识体系**：
- 引用的理论/文献/人物
- 常用类比和例子

---

## 输出要求

- 语言：中文
- 没有信息的维度：标注 `（原材料不足，建议追加相关内容）`
- 有原文依据的结论：加引号标注原话
- 输出结果直接用于生成 work.md，要求具体可执行，不要写"可能""倾向于"这类模糊表述
